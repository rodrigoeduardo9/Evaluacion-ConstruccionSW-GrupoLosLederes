Objetivo

Objetivo de la Evaluación

Evaluar la capacidad del equipo para trabajar de manera colaborativa utilizando Git y GitHub, mientras desarrollan, entrenan y evalúan modelos de Machine Learning para problemas de regresión y clasificación.

Dataset A: Regresión

Datos de precios de viviendas para modelos de regresión que predigan valores continuos.

Precios Regresión scikit-learn

Dataset B: Clasificación

Datos de historial de pacientes médicos para modelos de clasificación binaria.

Pacientes Clasificación Regresión Logística

Tecnologías

Python con bibliotecas estándar: scikit-learn, pandas, matplotlib/seaborn.

Python pandas scikit-learn
Estructura

Partes de la Evaluación

Cuatro partes principales que evalúan el trabajo colaborativo y las habilidades técnicas del equipo.

PARTE 1: Arquitectura del Repositorio y Git

30 minutos - 4 puntos

El equipo debe establecer el entorno de trabajo colaborativo.

  • Creación del Repositorio: Un líder de equipo debe crear un repositorio público en GitHub llamado Evaluacion-ConstruccionSW-GrupoX y agregar a los demás como colaboradores.
  • Estructura Inicial: Crear un archivo README.md con los nombres de los integrantes y un archivo .gitignore adecuado para Python.
  • Estrategia de Ramas: La rama main debe estar bloqueada para commits directos (requerir Pull Requests). Crear una rama de desarrollo llamada develop. Cada integrante debe trabajar en ramas de características (ej. feat/regresion-simple, feat/clasificacion).

PARTE 2: Modelos de Regresión Continua

60 minutos - 7 puntos

Usando el Dataset A. Cada modelo debe desarrollarse en su respectiva rama y luego integrarse a develop mediante un Pull Request (PR).

Regresión Lineal Simple

  • Seleccionar la variable independiente con mayor correlación con el precio
  • Entrenar el modelo
  • Graficar la línea de tendencia
  • Calcular MSE y R²

Regresión Lineal Múltiple

  • Utilizar al menos 3 variables independientes
  • Entrenar el modelo
  • Mostrar coeficientes obtenidos
  • Comparar R² con modelo simple

Regresión Polinómica

  • Transformación polinómica (grado 2 o 3)
  • Entrenar el modelo
  • Graficar curva resultante
  • Justificar ajuste

PARTE 3: Modelo de Clasificación

40 minutos - 6 puntos

Usando el Dataset B. Debe trabajarse en una rama feat/regresion-logistica.

  • Preprocesar los datos: Manejo de nulos y escalado de variables si es necesario
  • Entrenar modelo: Regresión Logística para predecir variable objetivo categórica (ej. "Enfermo" / "Sano")
  • Generar Matriz de Confusión
  • Calcular métricas: Accuracy, Precision y Recall
  • Realizar commit, push y abrir PR hacia develop

PARTE 4: Integración y Documentación

20 minutos - 3 puntos
  • Merge y Conflictos: Integrar todas las ramas a develop y finalmente hacer un PR hacia main
  • Resolver Conflictos: Si se generan conflictos (ej. en main.py o notebook), el equipo debe resolverlos adecuadamente dejando constancia en el historial
  • Documentación: Presentar informe final en PDF subido al aula virtual
